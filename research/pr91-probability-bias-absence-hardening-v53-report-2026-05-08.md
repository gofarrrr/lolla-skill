## Verdict

PR91 should be treated as an absence-rail enrichment, not a positive-affordance split.

The reread found real source richness across the probability/bias correction ring, but the extra material mostly did not pass the downstream transaction-identity test. The receiver would not usually need to use one new positive card while rejecting another positive card inside the same model. The useful delta is instead negative: make predictable misuse modes first-class so later packets can reject, defer, or warn without adding broad mental-model vocabulary.

## Decision Standard

The PR91 split question was not:

> Does the source contain more material?

Most sources do.

The operational question was:

> Would separating this material change a later receiver transaction: use, reject, defer, merge, or block?

Positive affordances were rejected unless the source supplied a distinct activation condition, evidence requirement, do-not-use boundary, treatment requirement, and likely receiver action.

Absence records were accepted when they prevented a likely overclaim or false promotion that the current positive affordance could otherwise make too easy to miss.

## Positive Split Decisions

No new positive affordances were added.

### `representativeness-heuristic`

Kept compressed as `representativeness-heuristic.similarity-vs-base-rate-check`.

Rejected positive split: analogy-as-communication.

Reason: the source supports analogy as a teaching and communication move, but the current Representativeness record explicitly excludes communication-only analogy. The corpus already has the adjacent owner `analogies-and-metaphors.structural-fit-transfer-test`. Duplicating that under Representativeness would blur ownership and make packet use less auditable.

### `base-rates`

Kept compressed as `base-rates.outside-view-reference-class-anchor`.

Rejected positive splits: Bayes update, reference-class forecasting, outside-view humility, domain dependence.

Reason: all are method details inside one transaction: anchor to a credible prior/reference class, test fit, then update with case-specific evidence.

### `conjunction-fallacy`

Kept compressed as `conjunction-fallacy.sequence-probability-stress-test`.

Rejected positive splits: MECE decomposition, weakest-link analysis, independent judgments, premortem questions.

Reason: these are treatment methods inside the same sequential-gate fragility card. Disjunctive failure risk is transaction-distinct in theory, but the source presents it mainly as a boundary and contrast, so it remains an absence/defer rail rather than a positive Conjunction card.

### `hindsight-bias`

Kept compressed as `hindsight-bias.predecision-record-for-learning-review`.

Rejected positive splits: predecision documentation, process/outcome separation, foreseeable-vs-foreseen review, alternative-path reconstruction.

Reason: all serve the same retrospective transaction: after an outcome is known, reconstruct the decision-time information state before judging process quality or extracting lessons.

No new absence record was added here because the existing two absence rails already block the critical failures: treating known outcome as foreseeability and running a postmortem without decision-time state.

### `wysiati`

Kept compressed as `wysiati.missing-evidence-denominator-audit`.

Rejected positive splits: denominator audit, disconfirming-case construction, information-asymmetry briefing audit.

Reason: these are all variants of the same transaction: confidence is being generated by a coherent visible story, so the receiver must name what is absent before commitment.

### `regression-to-the-mean`

Kept compressed as `regression-to-the-mean.baseline-before-causal-story`.

Rejected positive splits: intervention-overcredit check, reference-class baseline, tail-risk stress.

Reason: intervention overcredit and reference classes are treatment details inside the baseline-before-causal-story card. Tail risk is not a positive RTM affordance; it is a boundary guard because mean-reversion language can wrongly dismiss worst-case exposure.

## Added Absence Rails

Added five absence records:

- `demographic-stereotype-as-behavioral-proof`
- `single-optimistic-base-case-as-base-rate`
- `invented-step-probabilities-as-rigor`
- `generic-missing-information-boilerplate`
- `mean-reversion-as-tail-risk-dismissal`

These are intentionally negative affordance controls. They help future packet or decoder work avoid:

- treating demographic resemblance as behavioral proof;
- smuggling a single optimistic forecast in as a historical base rate;
- presenting invented step probabilities as rigor;
- replacing WYSIATI with vague "more information may exist" caution;
- using regression-to-the-mean language to downplay tail or worst-case risk.

## Why This Is Not Under-Extraction

The main risk in this ring was not thin extraction. The main risk was duplicate positive identity.

For example, Representativeness has source-backed communication material, but making it a Representativeness positive affordance would compete with the analogies record and weaken packet traceability. Likewise, WYSIATI has multiple named mitigation techniques, but they all execute the same receiver move: make missing evidence specific before confidence is warranted.

PR91 therefore preserves the corpus richness by making misuse boundaries explicit instead of inflating the positive affordance count.

## Follow-Up For Later Packet Stress

Future packet review should test whether absence rails are visible enough when rendered:

- Do vague WYSIATI cautions get blocked?
- Do invented probabilities get downgraded rather than treated as precision?
- Do broad probability cards avoid duplicate ownership when adjacent models are also nominated?
- Does RTM avoid dulling response to true tail risk or structural change?

If packet replay shows receivers repeatedly miss a transaction because an adjacent owner is not nominated, that would be evidence for a later positive split. PR91 did not find that evidence in the source layer alone.
