## Added Absence Records

### `conjunction-fallacy`

Added `disjunctive-failure-risk-as-conjunctive-sequence`.

Why: the source distinguishes conjunctive success chains from disjunctive failure structures. This is already present as a do-not-use boundary, but making it first-class as an absence record preserves the reject/defer rail when a future decoder sees a nomination that really belongs to disjunctive failure-path governance.

Added `expert-confidence-as-cumulative-risk-proof`.

Why: the current card already blocks story detail and detail-rich conjunctions, but not expert confidence as a substitute for independent cumulative-risk evidence. The source explicitly warns that expertise can create overconfidence and avoidance of objective cumulative-risk analysis.

### `category-decisions`

Added `high-status-category-without-independent-rationale`.

Why: the current record blocks shallow labels and buyer-frame-free category positioning. It did not block social contagion, where a high-status member's unverified categorization becomes the group's category. The source names this as an anti-pattern and gives independent rationale collection as mitigation.

### `mental-models-of-reality`

Added `single-familiar-frame-without-context-fit`.

Why: the current record blocks treating the map as reality and map-without-feedback. It did not first-class the source's warning that a familiar model can be misleading in a novel context. This is a decoder reject/defer guard for handy-frame transfer without context-fit checking.

### `evolutionary-pressure`

Added `threat-filter-communication-packaging` with `duplicate_of_existing_field`.

Why: the source supports croc-brain message packaging, but PR59 already rejected this as a positive split. The right action is not to promote a new evolutionary-pressure affordance. It is to record that this material should route to persuasion or communication cards unless the case also has a selection environment, reward structure, or repeated pressure.

### `international-negotiation-and-diplomacy-models`

Added `adversarial-countermove-simulation` with `duplicate_of_existing_field`.

Why: the source supports adversarial simulation and countermove thinking, but PR59 already rejected this as a positive split. Standalone countermove simulation is better owned by game-theory-payoffs, mental-simulation, or BATNA-style fallback analysis. The diplomacy card should stay focused on substance, signaling, stakeholder interpretation, and durable settlement alignment.

## Quality Rationale

This PR treats the one-affordance concern carefully.

The concern is real: one affordance can undercompress a rich source if multiple operational moves would produce different use/reject/defer/merge outcomes. But PR64 found that these reviewed candidates mostly did not need positive expansion. They needed better negative custody.

That means the enrichment is absence-first:

- preserve what the source warns against;
- stop duplicate positive splits from being rediscovered later;
- keep ownership boundaries visible;
- avoid making broad model language feel like judgment.

This is especially important before any future packet or decoder work. If a card gets nominated later, these absence records give the receiver a more precise reason to reject, defer, or route elsewhere.
